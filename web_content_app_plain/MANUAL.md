# YouTube Unified 사용 매뉴얼

## 1. 시작

1. `RUN_MENU.bat` 또는 `RUN_MENU.ps1`을 실행합니다.
2. 런처가 백엔드와 웹 서버를 자동으로 시작합니다.
3. 브라우저에서 `http://127.0.0.1:3000/` 화면이 열리면 사용합니다.

## 2. 먼저 설정할 것

왼쪽 `API 설정`에서 Gemini API Key를 입력하고 `저장`을 누릅니다.
`테스트`를 눌러 키가 정상인지 확인합니다.

- Gemini API Key: 스토리보드 생성과 이미지 생성에 필요합니다.
- 텍스트 모델: 대본 분석과 스토리보드 생성에 사용합니다.
- 이미지 모델: 씬 이미지를 생성할 때 사용합니다.
- 이미지 화풍: 모든 이미지에 적용할 스타일입니다.
- 커스텀 화풍: 직접 만든 영어 스타일 프롬프트입니다.

## 3. 스토리보드 생성 탭

새 영상을 기획하고 씬을 만드는 기본 작업 공간입니다.

1. `작업 이름`에 저장용 이름을 입력합니다.
2. `주제`에 만들 영상의 주제를 입력합니다.
3. 이미 대본이 있으면 `수동 대본`에 붙여넣습니다.
4. `스토리보드 생성`을 누르면 AI가 씬을 만듭니다.
5. AI 없이 문장만 나누려면 `대본만 씬 분할`을 누릅니다.
6. 씬별 `나레이션`과 `이미지 프롬프트`를 확인하고 수정합니다.
7. `전체 이미지 생성`으로 모든 씬 이미지를 생성합니다.
8. `MP4/WebM 내보내기`로 브라우저에서 영상 파일을 만듭니다.
9. 작업을 이어서 하려면 `프로젝트 저장`을 누릅니다.

보조 버튼:

- `JSON 내보내기`: 씬 데이터 전체를 파일로 저장합니다.
- `CSV 내보내기`: 표 형태로 씬 목록을 저장합니다.
- `비우기`: 현재 화면의 작업 내용을 초기화합니다.
- 씬의 `이미지` 버튼: 해당 씬 하나만 이미지를 다시 만듭니다.

## 4. 영상 자동화 탭

기존 유튜브 영상이나 로컬 영상 파일을 분석하는 탭입니다.

1. `프로젝트 이름`을 입력합니다.
2. `영상 경로/URL`에 유튜브 주소 또는 로컬 영상 경로를 입력합니다.
3. `자동화 시작`을 누릅니다.
4. 처리 로그를 확인합니다.
5. 후보 장면이 나오면 결과를 확인합니다.
6. `최종 영상 합성`으로 자막 포함 영상을 만듭니다.

고급 설정:

- `SALIENCY`: 중요한 장면 위주로 찾습니다.
- `UNIFORM`: 영상 전체에서 균일하게 장면을 찾습니다.
- `결과 수`: 후보 장면 개수입니다.
- `샘플링 간격`: 몇 초마다 영상을 검사할지 정합니다.
- `최소 간격`: 후보 장면끼리 떨어져야 하는 최소 시간입니다.

## 5. 자막 추출 탭

유튜브 영상에서 자막 텍스트를 가져오는 탭입니다.

1. `YouTube URL`에 영상 주소를 입력합니다.
2. `언어 코드`는 보통 `ko,en`을 사용합니다.
3. `자막 추출하기`를 누릅니다.
4. 결과가 나오면 `클립보드 복사`로 복사할 수 있습니다.
5. `스토리보드로 보내기`를 누르면 대본 입력칸으로 이동합니다.

추천 흐름:

1. 자막 추출
2. 스토리보드로 보내기
3. 스토리보드 생성
4. 이미지 생성
5. 영상 내보내기

## 6. 저장된 프로젝트 탭

브라우저에 저장된 작업을 관리합니다.

- `새로고침`: 저장 목록을 다시 읽습니다.
- `불러오기`: 저장된 프로젝트를 스토리보드 화면으로 가져옵니다.
- `삭제`: 저장된 프로젝트를 지웁니다.

저장 위치는 브라우저의 로컬 저장소입니다. 브라우저 데이터를 삭제하면 프로젝트도 사라질 수 있습니다.

## 7. 문제가 생겼을 때

- 스토리보드 생성이 안 되면 Gemini API Key를 저장하고 테스트합니다.
- 이미지 생성이 안 되면 Gemini 이미지 모델과 API Key를 확인합니다.
- 영상 자동화가 안 되면 런처에서 백엔드가 실행 중인지 확인합니다.
- MP4 저장이 안 되면 브라우저가 WebM으로 저장할 수 있습니다.
- 유튜브 자막이 없으면 해당 영상에 공개 자막이 없을 수 있습니다.

## 8. 권장 작업 순서

처음 사용하는 경우 아래 순서가 가장 안정적입니다.

1. API 설정 저장
2. 자막 추출 또는 직접 대본 입력
3. 스토리보드 생성
4. 씬 문장과 이미지 프롬프트 수정
5. 전체 이미지 생성
6. 프로젝트 저장
7. MP4/WebM 내보내기
