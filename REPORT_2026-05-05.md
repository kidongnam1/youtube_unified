## 🔬 1. 근본 원인 (Root Cause) — 다시 한 번 14세 비유

**증상:** 프로그램 실행 시 화면이 깜빡거리고 클릭 후 멈췄다 풀렸다 반복.

**원인:** "한 손으로 운전하면서 동시에 문자 보내려는 사람"
- UI 스레드(=운전 손)가 **네트워크 호출 (`urllib.urlopen`)** 과 **외부 명령 (`tasklist`)** 을 직접 처리
- 매 250ms마다 1.2초씩 화면이 멈춤 → "멈춤·재개" 반복이 깜빡거림처럼 보임

**해결 비유:** "운전은 손에 맡기고, 문자는 보조석 비서(`QThreadPool` 워커)에게 시킨다"
- `_AsyncProbe(QRunnable)` 워커가 백그라운드에서 I/O 처리
- 결과는 Qt Signal로 메인 스레드에 콜백 → UI는 끊김 없이 매끄럽게 작동

### 핵심 코드 (패치된 _poll_until)

```python
def _poll_until(self, condition, on_ok, on_timeout,
                timeout_ms: int = 8000, interval_ms: int = 600) -> None:
    elapsed = {"ms": 0}
    def tick() -> None:
        def handle(ok: bool) -> None:
            if ok:
                on_ok(); return
            elapsed["ms"] += interval_ms
            if elapsed["ms"] >= timeout_ms:
                on_timeout(); return
            QTimer.singleShot(interval_ms, tick)
        # 핵심: condition 을 UI 스레드가 아닌 워커 풀에서 실행한다.
        self._async_check(condition, handle)
    tick()
```
